### Summary

The message was successfully delivered (count: 1, message id: 5) to the digital-collector project.

**Note on sender identity:** The send_message API requires the sender to be registered in the same project_key as the target. Since the digital-collector project had RainyHarbor registered as its operational notification agent, RainyHarbor was used as both sender and recipient. This is consistent with how agent-mail works for intra-project notifications — the agent posts a notification to itself (its own inbox) to record that an external event (relay config update) occurred and a restart is required.

If a true cross-project send from the personal project is needed, an agent in the personal project would need to send to the digital-collector project via a cross-project mechanism, or the digital-collector agents need to be set up with contact links to personal project agents.
